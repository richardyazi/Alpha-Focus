# 聚宽选股程序说明
## 目的
    实现自动化股票筛选, 减少枯燥的人力劳动，提高效率。

## 运行环境
- 在聚宽投资研究环境运行(jupyter notebook)，指定Python 3.6.7版本
- 程序依赖聚宽提供的数据接口和API[聚宽API说明文档.md]
- 运行程序[key_select_stock.py]放在当前的[聚宽选股]子目录下，如期缺失该目录，请创建该目录

## 基础功能
### 标记关键K线
- 描述：在股票K线序列数据中，标记出关键K线:小阴小阳，十字星，B1;
- 输入：股票代码，K线周期，起始日期，结束日期
    - K线周期为1d, 1w, 1m, 3m, 6m, 1y
- 逻辑：
    - 通过股票代码，起止时间，获取股票日线数据(使用股票数据缓存池);
        - 注意：起止时间应结合K线周期以及文档[B1等K线形态的定义标准]各技术指标计算所需的历史数据长度和输入的起始时间动态计算得出;
            - 例如：K线周期为1d, 输入的起始时间是2024年1月1日，考虑到计算黄线需要取前114天的收盘价则起止时间应向前扩大至少114个交易日;
    - 将日线数据按照K线周期进行聚合，生成K线序列数据; 如果K先周期为1d, 则直接使用日线数据;
    - 参考文档[B1等K线形态的定义标准]内容，标记出关键K线:小阴小阳，十字星，B1;
- 输出：
    - 只输出起止时间之间的数据;
    - 类型DataFrame: index:交易时间, columns为['股票代码', '小阴小阳', '十字星', 'B1'];   格式如下：

| 交易时间       | 股票代码       | 小阴小阳 | 十字星 | B1   |
| :--------- | :------------- | :------: | :----: | :--- |
| 2025-06-01 | 000001.XSHE    |  False   | False  | False |
| 2025-06-02 | 000001.XSHE    |  False   | False  | False |
| 2025-06-03 | 000001.XSHE    |  False   | True   | True |
| 2025-06-04 | 000001.XSHE    |  False   | True   | False |

### 股票数据缓存池
- 描述：缓存股票数据供上层函数获取股票数据时调用，避免重复查询，减少API[get_price]调用次数; 
- 输入：股票代码，起始日期，结束日期
- 逻辑：
    - 若缓存中无该股票数据，则查询其上市日至最后一个完整交易日的历史数据并缓存。;
        - 最后一个完整交易日：数据已完整收录且交易日已结束的日期（不含当日以及非交易日）。
        - 历史数据的格式参考[get_price]函数，前复权，日线数据，不缓存非交易日数据以及当天数据(如果当天是交易日); 
    - 缓存数据按股票代码命名分文件和分目录存储，文件名为股票代码, 文件后缀为.csv,目录为[stock_cache/上市日期];
        - 缓存池[stock_cache] 在脚本文件所在目录下，若不存在，则创建该目录;
        - 上市日期为股票上市日期，只取年和月作为[stock_cache]的子目录名；
            - 例如: 000001.XSHE, 1991-04-03, 则 存放在 [stock_cache/1991-04/000001.XSHE.csv]
    - 如果缓存池中存在该股票代码的数据，则检查数据是否完整，如果不完整则更新数据；
        - 只更新到最后一个完整交易日，如果当天有交易，也不缓存当天数据；
    - 从股票池中获取股票在起始日期和结束日期之间的数据;如果起始日期和结束日期之间包含未结束的交易日或未来时间,则再调用[get_price]函数获取额外数据;
        - 合并缓存数据与实时数据，确保时间顺序正确且无重复。;
- 输出：

    - 类型DataFrame: index为交易时间，columns为['open', 'close', 'high', 'low', 'volume', 'money'];
    **示例：**

|  | open | close | high | low | volume | money |
|---|---|---|---|---|---|---|
| 2015-01-05 | 3566.09 | 3641.54 | 3669.04 | 3551.51 | 451198098.0 | 519849817448.0 |
| 2015-01-06 | 3608.43 | 3641.06 | 3683.23 | 3587.23 | 420962185.0 | 498529588258.0 |


## 使用方式
### 批量查询多个概念板块中符合B1的股票
- 描述：在板块中选出符合条件B1的股票;
- 输入：板块名，K线周期，起始日期，结束日期，数据量(count)
    - 板块名：可能是多个，多个板块名用|分隔, 只支持精准匹配，不支持模糊匹配；
    - K线周期 如果为空，默认1d;
    - 数据量 与 起始日期 二选一，不可同时使用, 必须大于 0. 表示返回 结束日期 之前 count 个交易数据, 包含 结束日期
        - 通过Count与结算时间计算起始时间需结合K线周期动态计算，不要考虑各技术指标所需要的历史数据长度(这个逻辑在标记关键K线函数中已经实现);
    - 结束日期 如果为空，默认当前日期;
- 逻辑：
    - 解析输入的板块名，获取板块编码列表;
    - 遍历所有板块编码，获取板块下面所有的股票代码;
    - 遍历所有股票代码，调用标记关键K线函数，获取标记结果;
        - 如果数据量不为空，需要根据结束日期和K线周期动态计算起始日期;
        - 如果数据量为空，则获取起始日期到结束日期之间的数据;        
    - 统计符合B1的股票代码，并输出;
- 输出：
    - 类型Tuple(符合B1的信息汇总, 股票的B1标记数据集)
        - 符合B1的信息汇总: 类型：dataframe, index:股票代码, columns: 最近一次的B1时间, B1出现的次数, 涉及的板块 ;
            - 涉及的板块 为板块名称，多个板块名称用|分隔，只包括输入中涉及的板块名；
            | 股票代码 | 最近一次B1时间 | B1出现次数 | 涉及的板块 |
            | :------- | :--------- | :--------- | :--------- | 
            | 000001   | 2025-06-09  | 1        | 板块A\|板块B |
            | 000002   | 2025-06-08  | 1        | 板块A |
            
        - 股票的B1标记数据集: 由所有相关股票的B1标记数据合并去重取得; 
            - 类型DataFrame: index:['交易时间','股票代码'], columns为['小阴小阳', '十字星', 'B1'];   格式如下：

            | 交易时间       | 股票代码       | 小阴小阳 | 十字星 | B1   |
            | :--------- | :------------- | :------: | :----: | :--- |
            | 2025-06-01 | 000001.XSHE    |  False   | False  | False |
            | 2025-06-02 | 000001.XSHE    |  False   | False  | False |
            | 2025-06-03 | 000001.XSHE    |  False   | True   | True |
            | 2025-06-04 | 000001.XSHE    |  False   | True   | False |
        


### 批量查询自选股票池中的符合B1的股票
- 描述: 从自选股中选出符合条件B1的股票; 自选股由用户维护，按组别进行管理，每个组别下可以包含多个股票代码;
- 输入：组别名, K线周期，起始日期，结束日期，数据量(count)
    - 组别名: 自选股组别名，多个组别名用|分隔，只支持精准匹配，不支持模糊匹配；
    - K线周期 如果为空，默认1d;
    - 数据量 与 起始日期 二选一，不可同时使用, 必须大于 0. 表示返回 结束日期 之前 count 个交易数据, 包含 结束日期
        - 通过Count与结算时间计算起始时间需结合K线周期动态计算，不要考虑各技术指标所需要的历史数据长度(这个逻辑在标记关键K线函数中已经实现);
    - 结束日期 如果为空，默认当前日期;
- 逻辑：
    - 获取自选股票池中的相应组别的股票代码; 
        - 自选股配置文件存放在代码文件所在目录的[config]文件夹中，命名为[stocks_pool.json]; 
        - 若配置文件不存在，应创建默认配置文件，并给出示例模版，方便用户在此基础上完善;        
    - 应在运行过程中显示命中了哪些自选股的组别以及该组别下的股票数量;
    - 用户配置的股票代码其他形式的股票代码, 需要转换成聚宽代码形式; 可以使用聚宽提供的[normalize_code]函数
    - 遍历所有股票代码，调用标记关键K线函数，获取标记结果;
        - 如果数据量不为空，需要根据结束日期和K线周期动态计算起始日期;
        - 如果数据量为空，则获取起始日期到结束日期之间的数据;
    - 统计符合B1的股票代码，并输出;
- 输出：
    - 类型Tuple(符合B1的信息汇总, 股票的B1标记数据集)
        - 符合B1的信息汇总: 股票代码, 最近一次的B1时间, B1出现次数, 涉及的自选组;

            | 股票代码 | 最近一次B1时间 | B1出现次数 | 涉及的自选组 |
            | :------- | :------------ | :--------- | :----------- |
            | 000001   | 2025-06-09    | 1          | 组A\|组B     |
            | 000002   | 2025-06-08    | 1          | 组C          |
            
        - 股票的B1标记数据集: 由所有相关股票的B1标记数据合并去重取得; 
            - 类型DataFrame: index:['交易时间','股票代码'], columns为['小阴小阳', '十字星', 'B1'];   格式如下：

            | 交易时间       | 股票代码       | 小阴小阳 | 十字星 | B1   |
            | :--------- | :------------- | :------: | :----: | :--- |
            | 2025-06-01 | 000001.XSHE    |  False   | False  | False |
            | 2025-06-02 | 000001.XSHE    |  False   | False  | False |
            | 2025-06-03 | 000001.XSHE    |  False   | True   | True |
            | 2025-06-04 | 000001.XSHE    |  False   | True   | False |



### 全量扫描符合B1的股票
- 描述: 扫描所有股票，选出符合条件B1的股票;

- 输入：K线周期，起始日期，结束日期，数据量(count)
    - K线周期 如果为空，默认1d;
    - 数据量 与 起始日期 二选一，不可同时使用, 必须大于 0. 表示返回 结束日期 之前 count 个交易数据, 包含 结束日期
        - 通过Count与结算时间计算起始时间需结合K线周期动态计算，不要考虑各技术指标所需要的历史数据长度(这个逻辑在标记关键K线函数中已经实现);
    - 结束日期 如果为空，默认当前日期;

- 逻辑：
    - 获取所有股票代码; 
        - 可以使用聚宽提供的[get_all_securities]函数获取所有股票代码;
    - 默认去掉ST,创业板以及北交所股票;
    - 遍历所有股票代码，调用标记关键K线函数，获取标记结果;

- 输出：
    - 类型Tuple(符合B1的信息汇总, 股票的B1标记数据集)
        - 符合B1的信息汇总: 股票代码, 最近一次的B1时间, B1出现次数 ;

            | 股票代码 | 最近一次B1时间 | B1出现次数 |
            | :------- | :--------- | :--------- | 
            | 000001   | 2025-06-09  | 1        |
            | 000002   | 2025-06-08  | 1        |
            
        - 股票的B1标记数据集: 由所有相关股票的B1标记数据合并去重取得; 
            - 类型DataFrame: index:['交易时间','股票代码'], columns为['小阴小阳', '十字星', 'B1'];   格式如下：

            | 交易时间       | 股票代码       | 小阴小阳 | 十字星 | B1   |
            | :--------- | :------------- | :------: | :----: | :--- |
            | 2025-06-01 | 000001.XSHE    |  False   | False  | False |
            | 2025-06-02 | 000001.XSHE    |  False   | False  | False |
            | 2025-06-03 | 000001.XSHE    |  False   | True   | True |
            | 2025-06-04 | 000001.XSHE    |  False   | True   | False |

### 根据股票的B1标记数据，统计概念板块B1出现的情况
- 描述: 基于已知的股票B1标记数据集合或者实时计算相应的股票B1标记数据集合，在此基础上统计相关概念板块的B1出现的情况;

- 输入：股票的B1标记数据, 板块名, 组别名, K线周期，起始日期，结束日期，数据量(count)
    - [输入：股票的B1标记数据]的格式参考[标记关键K线]函数的输出;
    - 使用方式1: 只输入[股票的B1标记数据],再此基础上直接统计;默认None
    - 使用方式2: 输入[板块名], K线周期，起始日期，结束日期，数据量(count),先调用[批量查询多个概念板块中符合B1的股票]函数获取股票的B1标记数据, 再此基础上统计;
        - 板块名: 可能是多个，多个板块名用|分隔, 只能精确查找, 不支持模糊查找; 默认None
    - 使用方式3: 输入[组别名], K线周期，起始日期，结束日期，数据量(count),先调用[批量查询自选股票池中的符合B1的股票]函数获取股票的B1标记数据, 再此基础上统计;
        - 组别名: 可能是多个，多个组别名用|分隔, 只能精确查找, 不支持模糊查找; 默认None
    - 使用方式4: [股票的B1标记数据,板块名,组别名] 均无输入, 先调用[全量扫描符合B1的股票]函数获取股票的B1标记数据, 再此基础上统计;
    - K线周期 如果为空，默认1d;
    - 数据量 与 起始日期 二选一，不可同时使用, 必须大于 0. 表示返回 结束日期 之前 count 个交易数据, 包含 结束日期
        - 通过Count与结算时间计算起始时间需结合K线周期动态计算，不要考虑各技术指标所需要的历史数据长度(这个逻辑在标记关键K线函数中已经实现);
    - 结束日期 如果为空，默认当前日期;
- 逻辑：
    - 如何获取股票的B1标记数据: 
        - 当[股票的B1标记数据]不为None时，直接使用该数据;
        - 当[板块名]不为None时，先调用[批量查询多个概念板块中符合B1的股票]函数获取股票的B1标记数据;
        - 当[组别名]不为None时，先调用[批量查询自选股票池中的符合B1的股票]函数获取股票的B1标记数据;
        - 当[板块名]和[组别名]均为None时，先调用[全量扫描符合B1的股票]函数获取股票的B1标记数据;
    - 统计逻辑：
        - 遍历股票的B1标记数据(B1列为True)，获取股票所属的概念板块，并统计每个概念板块中B1出现的累计次数和B1出现的股票数;
            - 注意：一只股票可能属于多个概念版本，需要在各概念版本中都进行统计;
        - 遍历股票的B1标记数据(B1列为True)，获取B1出现的交易时间，并统计每个交易日B1出现的次数;
- 输出：
    - 类型Tuple(概念板块统计, 交易日统计)
        - 概念板块统计：类型DataFrame: 板块代码, 板块名称, 出现B1的股票数量, B1出现的总次数 ;
        - 交易日统计：类型DataFrame: 交易日期, B1出现的次数 ;
        - 格式如下(确保列顺序)：
            - 概念板块统计： 

                | 板块代码  | 板块名称         | 出现B1的股票数量 | B1出现的总次数 |
                | :-------- | :-------------  | :--------------- | :------------|
                | SC0012    | 人工智能        | 12               | 15            |
                | SC0034    | 新能源汽车      | 20               | 26            |
                | SC0056    | 风电概念        | 8                | 10            |

            - 交易日统计：           

                | 交易日期  | B1出现的次数 |
                | :-------- | :------------|
                | 2025-06-09  | 15            |
                | 2025-06-08  | 10            |
                | 2025-06-07  | 8            |
